1. 개요 — 스키마 생성 방식과 host 허브 구조

스키마는 Hibernate 가 자동 생성한다(spring.jpa.hibernate.ddl-auto: update). 별도 마이그레이션 도구(Flyway/Liquibase)는 쓰지 않으므로, 본 문서가 곧 사람이 읽는 스키마다. update 는 컬럼 추가만 하고 삭제·타입 변경은 하지 않는다.

전체 구조의 핵심은 host(도메인) 가 모든 도메인 데이터의 중심 키라는 점이다. 도메인 하나가 설정(domain_config)·스펙(spec_record)·검출 카탈로그(discovered_endpoint)·문서화 API 인벤토리(documented_api)·최신 결과(scan_result)·워터마크(watermark)·분류 override(domain_classification_config)로 펼쳐진다. 도메인과 무관한 전역 테이블은 둘뿐이다 — 전역 분류 기본값(classification_config, 단일 행)과 정적 분류 규칙 집합(static_classify_rule).

★ 실제 외래키(FK) 제약은 단 하나뿐 엔티티에 JPA 연관(@ManyToOne/@OneToOne)이 하나도 선언돼 있지 않다. 따라서 테이블 간 관계는 대부분 host 문자열을 공유하는 논리 FK(애플리케이션 레벨)이며 DB FK 제약으로 강제되지 않는다. 물리 FK 가 실제로 생성되는 곳은 @ElementCollection 자식 테이블인 domain_hostnames → domain_config 한 곳뿐이다.

2. ER Diagram — host 를 허브로 한 방사형 관계

domain_config 를 중심으로 1:1 위성(왼쪽)과 1:N 자식(오른쪽)이 host 로 연결된다. 실선 굵은 파랑은 실 FK, 가는 회색은 논리 FK(host 공유), 보라 점선은 전역→도메인 상속(앱 레벨 병합)이다.

3. 타입 매핑 컨벤션 (JPA → SQL)

이식성을 위해 벤더 전용 JSON 타입(PostgreSQL JSONB)을 쓰지 않는다. JSON 은 모두 String + @Column(columnDefinition="text") 로 저장하고 애플리케이션이 Jackson 으로 직렬화한다(H2/PostgreSQL 동일 동작). 아래 §4 의 모든 컬럼 타입은 이 규칙을 따른다.

Java / JPAH2PostgreSQLnullable 기본
String (length 미지정)VARCHAR(255)varchar(255)nullable
String + columnDefinition="text" (JSON)texttextnullable
@Lob byte[] (現 미사용)BLOBoid(PostgreSQL 대용량 객체 식별자)nullable
InstantTIMESTAMP(6)timestamp(6)nullable (UTC)
long / int / boolean (primitive)BIGINT / INTEGER / BOOLEANbigint / integer / booleanNOT NULL
Long / Double (wrapper)BIGINT / DOUBLEbigint / double precisionnullable
@Enumerated(STRING) enumVARCHAR(255)varchar(255)nullable
nullable 은 타입이 결정한다 static_classify_rule(kind/rule_value 에 @Column(nullable=false))를 제외하면 어떤 필드에도 @Column(nullable=false) 가 없다. 따라서 그 나머지는 원시 타입(long/int/boolean)만 NOT NULL, 래퍼/객체 타입(String/Long/Double/Instant/enum/byte[])은 nullable 이다. @Id 컬럼은 PK 라 묵시적 NOT NULL. 또한 enabled=true·state="idle" 등 필드 기본값은 Java 초기화자일 뿐 SQL DEFAULT 절이 아니다(유일한 예외 = scan_result.total_dropped integer default 0).

4. 테이블별 컬럼 명세 (10 테이블)

표기: PK 기본키 · NN NOT NULL · null nullable. JSON 컬럼은 모두 PostgreSQL text(D40 실측 확정).

4.2 domain_hostnames — 도메인↔엣지 hostname 컬렉션

컬럼SQL 타입키/널의미

별도 PK 컬럼 없음 — (host, hostname) 조합이 행을 이룬다.
